【演绎推理】
从一般性原理推导出具体结论

核心结构：
大前提（普遍规律）+ 小前提（具体情况）→ 结论

经典三段论：
大前提：所有A都是B
小前提：C是A
结论：所以C是B

示例：
大前提：所有哺乳动物都需要呼吸
小前提：鲸鱼是哺乳动物
结论：所以鲸鱼需要呼吸

演绎推理步骤：

1. 识别适用的普遍规律
   - 这个问题涉及什么原理/定律？
   - 已知的通则是什么？
   - 理论框架如何说？

2. 确认具体情况
   - 当前的具体情况是什么？
   - 是否满足前提条件？
   - 有无例外情况？

3. 验证前提的真实性
   - 大前提是否可靠？
   - 小前提是否准确？
   - 逻辑关系是否成立？

4. 推导结论
   - 按照逻辑规则推导
   - 确保推理的必然性
   - 检查是否有逻辑谬误

5. 检验结论的有效性
   - 结论是否符合逻辑？
   - 是否与其他已知事实矛盾？
   - 是否需要附加条件？

常见演绎推理模式：

1. 肯定前件式（Modus Ponens）
   如果P，则Q
   P成立
   所以Q成立

2. 否定后件式（Modus Tollens）
   如果P，则Q
   Q不成立
   所以P不成立

3. 选言推理
   P或Q
   非P
   所以Q

4. 假言推理链
   如果P，则Q
   如果Q，则R
   所以如果P，则R

演绎推理的优势：
✓ 结论必然性强
✓ 逻辑严密
✓ 可以用于证明

演绎推理的局限：
✗ 依赖前提的正确性
✗ 不能产生新知识（结论包含在前提中）
✗ 现实中的前提常常不完全确定

常见逻辑谬误要避免：
✗ 肯定后件：如果P则Q，Q成立，所以P成立（错误）
✗ 否定前件：如果P则Q，P不成立，所以Q不成立（错误）
✗ 偷换概念：前后使用同一词汇但含义不同
✗ 循环论证：结论本身作为前提

应用场景：
- 数学证明
- 法律推理
- 科学假说验证
- 技术问题诊断
- 规则系统应用

输出格式：
- 大前提：[普遍规律]
- 小前提：[具体情况]
- 推理过程：[逻辑链条]
- 结论：[必然推出的结论]
- 置信度：[前提可靠性评估]
