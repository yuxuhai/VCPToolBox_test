【安全测试】
发现和修复安全漏洞

安全测试类型：
1. 漏洞扫描
   - 已知漏洞
   - 依赖安全检查
   - 配置审计

2. 渗透测试
   - 模拟攻击
   - 权限绕过
   - 数据泄露

3. 代码审计
   - 静态分析
   - 安全编码规范
   - 敏感信息泄露

常见安全测试：
• SQL注入测试
  ```
  ' OR '1'='1
  1'; DROP TABLE users--
  ```

• XSS测试
  ```html
  <script>alert('XSS')</script>
  <img src=x onerror=alert('XSS')>
  ```

• CSRF测试
  - 检查CSRF Token
  - 验证Referer
  - SameSite Cookie

• 认证授权测试
  - 弱密码
  - 暴力破解
  - 权限越权
  - Token安全

• 敏感数据测试
  - 传输加密(HTTPS)
  - 存储加密
  - 日志脱敏

测试工具：
→ OWASP ZAP：漏洞扫描
→ Burp Suite：渗透测试
→ npm audit：依赖检查
→ SonarQube：代码审计
→ Snyk：安全扫描

安全检查清单：
□ 输入验证
□ 输出编码
□ 认证机制
□ 授权控制
□ 加密传输
□ 敏感数据保护
□ 日志审计
□ 错误处理

OWASP Top 10：
1. 注入
2. 失效的身份认证
3. 敏感数据泄露
4. XML外部实体(XXE)
5. 失效的访问控制
6. 安全配置错误
7. 跨站脚本(XSS)
8. 不安全的反序列化
9. 使用含有已知漏洞的组件
10. 不足的日志记录和监控
