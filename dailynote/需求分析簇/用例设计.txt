【用例设计】
完整描述系统行为

用例结构：
1. 基本信息
   - 用例名称
   - 参与角色
   - 前置条件
   - 后置条件

2. 主流程
   - 正常操作步骤
   - 系统响应
   - 用户反馈

3. 异常流程
   - 错误情况
   - 边界case
   - 异常处理

4. 补充说明
   - 业务规则
   - 非功能需求
   - 注意事项

用例示例：
```
用例：用户登录

角色：未登录用户

前置条件：
- 用户已注册
- 网络正常

主流程：
1. 用户打开登录页
2. 输入账号和密码
3. 点击登录按钮
4. 系统验证信息
5. 跳转到首页

异常流程：
A1. 账号不存在
    - 提示"账号未注册"
A2. 密码错误
    - 提示"密码错误"
    - 3次失败锁定账号
A3. 网络超时
    - 提示"网络异常，请重试"

后置条件：
- 用户登录状态已建立
- 生成session/token
```

设计原则：
✓ 完整性：覆盖所有场景
✓ 清晰性：步骤明确无歧义
✓ 可测性：可转化为测试用例
✓ 用户视角：从用户出发

用例覆盖：
□ 正常流程
□ 异常处理
□ 边界条件
□ 并发情况
□ 性能要求

输出：
- 用例清单：[所有用例]
- 详细用例：[每个用例的完整描述]
- 用例关系：[依赖和顺序]
- 优先级：[重要程度]
