【需求理解】
准确把握用户真实需求

理解层次：
1. 表面需求（用户说的）
   - 字面描述
   - 初步想法

2. 真实需求（用户要的）
   - 实际问题
   - 核心诉求

3. 潜在需求（用户未意识到的）
   - 延伸需求
   - 体验优化

分析方法：
• 5W2H追问
  - What：要做什么？
  - Why：为什么做？（核心）
  - Who：谁使用？
  - When：何时用？
  - Where：在哪用？
  - How：怎么做？
  - How much：成本/规模？

• 场景还原
  用户在XX情况下，想要XX，以实现XX

• 痛点挖掘
  - 现有方案的问题
  - 操作中的摩擦点
  - 未被满足的期待

需求澄清：
✓ 与需求方反复确认
✓ 用原型/草图验证理解
✓ 识别隐含假设
✓ 明确优先级

需求分类：
→ 功能需求：系统要做什么
→ 性能需求：要做到什么程度
→ 体验需求：使用感受如何
→ 约束条件：技术/时间/资源限制

警惕：
✗ 接受模糊需求
✗ 只听表面不问为什么
✗ 假设用户意图
✗ 过度解读

输出：
- 需求描述：[用户想要什么]
- 核心目标：[为什么要这个]
- 使用场景：[何时何地怎么用]
- 成功标准：[如何算成功]
